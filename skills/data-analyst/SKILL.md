# Data Analyst

## 触发条件
- 数据、指标、漏斗、留存、显著性、A/B测试、cohort
- "深度分析数据"、"分析用户行为/留存/转化"、"找根因"

## 角色定位
从数据中提取洞察，指导业务决策。透明化、可验证、强业务关联。

## 工作方式
1. 先问清业务问题，再分析数据
2. **所有数据处理必须附逻辑说明**：计算公式、聚合方法、表合并策略
3. 给数据时同步给解读和行动建议
4. 区分相关性 vs 因果性

## 核心框架

### 三层分析模型
| 层级 | 问题 | 方法 |
|------|------|------|
| 描述性(What) | 现状如何？ | 统计、分布、趋势 |
| 诊断性(Why) | 为什么？ | 漏斗、路径、分层、交叉 |
| 建议性(How) | 怎么做？ | 根因定位、策略建议、监控体系 |

### OSM 模型
- **O**bjective：业务目标是什么？
- **S**trategy：达成策略是什么？
- **M**easure：衡量指标是什么？

## 最佳实践（8条）

### 1. 数据清洗：标记 vs 过滤
| 数据类型 | 处理方式 | 场景 |
|---------|---------|------|
| 噪音数据 | **标记** | 不同口径需不同处理时 |
| 无效数据 | **过滤** | 确定不影响任何分析时 |
| 测试数据 | **隔离** | 单独标记便于对比 |

### 2. 指标定义：三层确认
```
指标 = 业务语义 + 计算逻辑 + 统计口径

例：日均有效生成次数
- 业务语义：排除系统默认，仅模板生成
- 计算逻辑：有效会话数 / 天数
- 统计口径：按会话去重（非用户累加）
```

### 3. 用户分群：先分层后计算
```
1. 明确分群目的
2. 确定有效样本（排除0次等无效样本）
3. 基于有效样本计算分布
4. 应用分层阈值
```
**陷阱**：包含无效样本算分位数会严重失真

### 4. 用户状态：精准区分
| 状态 | 定义 | 策略 |
|------|------|------|
| 沉默用户 | 进入未激活 | 降低门槛 |
| 流失用户 | 用过不用 | 召回激励 |
| 回流用户 | 重新使用 | 复制成功 |

### 5. 对比分析：四关对齐
- [ ] 时间周期（天数一致？）
- [ ] 样本范围（新增vs存量？）
- [ ] 定义一致（口径相同？）
- [ ] 外部因素（节假日/活动？）

### 6. 数据洞察：业务验证
```
数据发现 → 业务解释 → 验证确认

例：外部客户转化率低
- 数据：外部15% vs 内部55%
- 解释：是否受权限限制？
- 验证：仅10家开白名单
- 结论：非产品问题，是触达限制
```

### 7. 交付物：受众分层
| 受众 | 形式 | 重点 |
|------|------|------|
| 决策者 | 1页摘要 | 核心结论+数字 |
| 执行者 | 操作清单 | 具体到人+时间 |
| 分析师 | 方法论 | 可复现+假设 |

### 8. 检查点机制
```
阶段1：数据准备 → 确认数据范围
阶段2：核心指标 → 确认计算口径
阶段3：深度分析 → 确认分析方向
阶段4：策略建议 → 确认落地可行性
```

## 核心分析方法

### 1. 行为路径分析
```python
# 聚合行为序列 → 定义分层规则 → 统计活跃天数
user_seq = df.groupby('user_id')['action'].apply(list)
def classify(actions):
    if 'purchase' in actions: return '深度'
    elif 'view' in actions: return '中度'
    else: return '浅度'
```

### 2. 转化漏斗分析
```python
# 统计各阶段去重用户 → 计算转化率/流失率
funnel = {
    'enter': df[df['action']=='enter']['user_id'].nunique(),
    'view': df[df['action']=='view']['user_id'].nunique(),
    'convert': df[df['action']=='convert']['user_id'].nunique()
}
# 转化率 = 当前阶段 / 上一阶段
# 流失率 = 1 - 转化率
```

### 3. 留存分析
```python
# 计算用户首次日期 → 合并回原表 → 计算间隔天数 → 统计留存率
first = df.groupby('user_id')['date'].min().reset_index()
df = df.merge(first, on='user_id')
df['days_since'] = (df['date'] - df['first_date']).dt.days
retention = df.groupby('days_since')['user_id'].nunique() / total_users
```

### 4. RFM用户分层
```python
# 计算R(近度)F(频度)M(金额) → 基于阈值分层
rfm = df.groupby('user_id').agg({
    'date': lambda x: (today - x.max()).days,  # R
    'action': 'count',  # F
    'value': 'sum'  # M
})
# 分层规则：R≤7且F≥5=核心用户；R≤30且F≥3=重要用户...
```

### 5. 数据与反馈融合
```python
# 从定量数据识别目标用户 → 关联反馈表 → 模式匹配 → 计算置信度
target_users = identify_low_retention(df)
feedback = feedback_df[feedback_df['user_id'].isin(target_users)]
# 置信度：≥3条不同用户=高；1-2条=中；无=低
```

## 数据处理规范

### 指标计算说明
- **去重计数**：`nunique()` 独立用户数
- **条件计数**：先筛选再 `len()`
- **比率计算**：分子/分母×100%，明确分母口径

### 数据聚合方法
```python
# 分组聚合
df.groupby('date').agg({'user_id': 'nunique', 'revenue': 'sum'})

# 滚动聚合
df['revenue_7d'] = df['revenue'].rolling(window=7).mean()
```

### 数据表合并
```python
# 一对一/一对多合并
merged = df1.merge(df2[['key', 'col']], on='key', how='left')

# 多对多需谨慎：先对一方去重
users_deduped = users_df.drop_duplicates(subset=['user_id'])
```

### 数据清洗
```python
# 筛选
active = df[df['last_login'] >= '2024-01-01']
high_value = df[(df['spent'] > 1000) & (df['orders'] >= 5)]

# 去重：保留最新记录
df_sorted = df.sort_values('timestamp', ascending=False)
df_deduped = df_sorted.drop_duplicates(subset=['user_id'], keep='first')

# 缺失值填充
df['age'] = df['age'].fillna(df['age'].median())
df['city'] = df['city'].fillna('未知')
```

### 统计显著性检验
```python
from scipy import stats

# 两比例z检验
def z_test(conv_a, total_a, conv_b, total_b):
    p_a, p_b = conv_a/total_a, conv_b/total_b
    p_pool = (conv_a + conv_b) / (total_a + total_b)
    se = (p_pool * (1-p_pool) * (1/total_a + 1/total_b)) ** 0.5
    z = (p_a - p_b) / se
    p_value = 2 * (1 - stats.norm.cdf(abs(z)))
    return z, p_value
# p < 0.05 表示差异显著
```

## 使用流程

1. **明确目标**：业务背景、要解决的问题、是否有反馈数据可融合
2. **数据探索**：加载、质量检查、基础统计、识别异常
3. **深度分析**：根据目标选择框架（留存→行为+分层；转化→漏斗；价值→RFM）
4. **融合反馈**：收集相关反馈、印证数据发现、识别数据无法解释的现象
5. **生成洞察**：提炼3-5个核心洞察，每个配数据支撑+反馈印证+建议
6. **结果呈现**：Executive summary置顶、分层展示、可执行清单

### 数据处理逻辑验证标准

任何数据处理操作必须说明：
1. **指标计算**：名称含义、计算公式、数据来源、统计口径
2. **数据聚合**：分组维度、聚合函数、筛选条件
3. **表合并**：主表/关联表、关联键、合并方式、预期行数变化

**标准表述**：
```
指标：次日留存率
计算：第1日活跃用户数 / 首日新增用户数 × 100%
来源：user_activity表
聚合：按first_date和days_since_first分组，user_id去重
筛选：first_date = '2024-01-01'
```
