# 麦肯锡式分析思路

咨询公司标准分析框架，适用于商业策略报告。

## 触发条件

- 需要结构化商业问题分析
- 需要向管理层汇报分析结论
- 需要构建有说服力的商业论证

## 核心框架

### 1. MECE 原则

**Mutually Exclusive, Collectively Exhaustive**
（相互独立，完全穷尽）

```
问题拆解示例：

"如何提升公司营收？"

❌ 非MECE：
├── 增加销售
├── 提高价格
├── 开拓新市场
└── 降低成本  ← 成本和营收不是同一维度

✅ MECE（按业务线）：
├── 产品线A营收提升
│   ├── 提升单价
│   └── 增加销量
├── 产品线B营收提升
│   ├── 提升单价
│   └── 增加销量
└── 新产品线开发
    ├── 已有市场新产品
    └── 新市场新产品
```

### 2. 问题拆解方法

#### 逻辑树（Issue Tree）

```
核心问题：AI 对就业市场的影响是什么？

├── 供给侧影响（劳动者角度）
│   ├── 岗位替代风险
│   │   ├── 高暴露度岗位识别
│   │   └── 受影响人数估算
│   ├── 技能需求变化
│   │   ├── 新兴技能需求
│   │   └── 传统技能贬值
│   └── 薪资结构变化
│       ├── 高暴露岗位薪资趋势
│       └── 低暴露岗位薪资趋势
│
├── 需求侧影响（企业角度）
│   ├── 招聘需求变化
│   │   ├── AI相关岗位增长
│   │   └── 传统岗位减少
│   └── 人效提升
│       ├── 单位产出变化
│       └── 人均产值变化
│
└── 结构性影响（市场角度）
    ├── 行业结构变化
    ├── 区域分布变化
    └── 职业流动趋势
```

#### 假设树（Hypothesis Tree）

```
核心假设：AI 将加剧就业市场两极分化

├── H1: 高技能高薪资岗位需求增加
│   ├── 证据1: AI开发岗位增长
│   ├── 证据2: AI应用专家薪资上涨
│   └── 数据验证: 招聘平台数据
│
├── H2: 低技能重复性岗位减少
│   ├── 证据1: 客服岗位下降
│   ├── 证据2: 数据录入岗位下降
│   └── 数据验证: 就业统计年鉴
│
└── H3: 中间层岗位受挤压最严重
    ├── 证据1: 中等技能岗位增长停滞
    ├── 证据2: 中等薪资群体流动性下降
    └── 数据验证: 职业流动数据
```

### 3. 金字塔原理

**结论先行，以上统下，归类分组，逻辑递进**

```markdown
# 执行摘要（金字塔顶端）

**核心结论**: AI 技术将导致中国就业市场出现明显的两极分化，
预计 1.47 亿中等暴露度中等薪资劳动者面临转型压力。

## 关键发现（支撑论点）

1. **岗位替代风险集中在中间层**
   - 高暴露度职业中，薪资中位数群体占比62%
   - 346个职业中，中等暴露度（0.4-0.7）就业人数最多

2. **技能溢价显著扩大**
   - AI开发岗位薪资增速是普通岗位的2.3倍
   - 低技能岗位薪资增长停滞

3. **区域差异明显**
   - 一线城市AI岗位集中度高出三四线城市5倍
   - 传统产业集中地区转型压力更大

## 建议行动（基于发现）

1. 短期：识别高风险岗位，启动技能评估
2. 中期：投资AI技能培训，建立内部转岗机制
3. 长期：调整人才战略，储备AI复合型人才
```

## 分析流程

```
Step 0: 定义问题
   └── 用一句话明确核心问题
   └── 确定分析边界（时间/范围/深度）

Step 1: 拆解问题
   └── 构建MECE逻辑树
   └── 识别关键分支（80/20原则）

Step 2: 建立假设
   └── 为关键分支提出假设
   └── 按影响力排序假设优先级

Step 3: 验证假设
   └── 收集数据验证/证伪假设
   └── 更新假设树（剔除不成立假设）

Step 4: 综合洞察
   └── 提炼SO WHAT（那又怎样）
   └── 构建金字塔结构结论

Step 5: 提出建议
   └── 基于洞察提出 actionable 建议
   └── 明确责任人和时间表
```

### 80/20 分析（帕累托）

```python
# 识别关键驱动因素
import pandas as pd

# 按影响程度排序
df_sorted = df.sort_values('impact', ascending=False)

# 计算累计占比
df_sorted['cumulative_pct'] = df_sorted['impact'].cumsum() / df_sorted['impact'].sum()

# 识别关键20%（通常累计贡献80%）
key_factors = df_sorted[df_sorted['cumulative_pct'] <= 0.8]

print(f"关键因素数量: {len(key_factors)} / {len(df)} ({len(key_factors)/len(df)*100:.1f}%)")
print(f"关键因素贡献: {key_factors['impact'].sum() / df['impact'].sum() * 100:.1f}%")
```

## 商业洞察提炼

### SO WHAT 测试

每个发现必须通过 SO WHAT 测试：

| 发现 | SO WHAT | 洞察 |
|------|---------|------|
| AI暴露度与薪资正相关r=0.39 | 那又怎样？ | 高薪资群体通过掌握AI工具形成护城河，低薪资群体面临双重挤压 |
| 1.47亿人处于中等暴露区间 | 那又怎样？ | 这是最大的转型风险群体，需要政策重点关注 |
| 年轻人入场率下降14% | 那又怎样？ | 代际技能断层风险，未来劳动力供给质量可能下降 |

### 洞察写作公式

```
【数据发现】+ 【深层含义】+ 【行动暗示】

示例：
"数据显示高AI暴露职业中低薪群体占比达35%（发现），
意味着技术应用红利未普惠到基层劳动者（含义），
企业在AI转型中需配套技能培训和薪酬调整机制（行动）。"
```

## 禁止事项

- 不堆砌数据：每个图表必须有明确结论支撑
- 不混淆相关与因果：明确标注"相关"vs"导致"
- 不忽略反证：主动呈现与假设矛盾的证据
- 不空泛建议：每条建议必须 actionable（可执行、可衡量）
