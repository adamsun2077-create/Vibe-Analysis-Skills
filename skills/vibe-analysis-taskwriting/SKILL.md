# Vibe Analysis 任务书写法

把模糊好奇心变成精确分析指令的核心方法论。

## 触发条件

- 用户想做一个数据分析但表述模糊（"分析一下这个数据"）
- 需要明确分析目标和范围
- 需要撰写结构化的分析任务书

## 四要素结构

每个完整的分析任务书必须包含：

```
1. 分析目标（Why）
   - 核心问题：你想通过数据回答什么问题？
   - 决策价值：答案将如何影响行动？

2. 待验证假设（What）
   - H1/H2/H3 格式编号
   - 每条假设必须绑定数据来源
   - 假设必须可证伪（有明确的验证标准）

3. 分析步骤（How）
   - Step1: 数据校验（记录数/缺失率/一致性）
   - Step2: 描述性分析（分布/趋势/异常）
   - Step3: 假设验证（统计检验/相关性）
   - Step4: 探索性发现（开放但记录）
   - Step5: 产物输出（洞察/报告/看板）

4. 产物要求（Deliverable）
   - insights.md: 3-5条核心洞察（面向非技术读者）
   - report.md: 完整分析报告（技术+业务）
   - dashboard.html: 可视化看板（适合截图传播）
```

## 假设写法规范

### 格式模板

```markdown
## 待验证假设

**H1: [描述预期关系]**
- 数据来源: [具体文件/表名]
- 验证方法: [统计方法]
- 成功标准: [具体数值阈值]

**H2: [描述预期关系]**
- 数据来源: [具体文件/表名]
- 验证方法: [统计方法]
- 成功标准: [具体数值阈值]
```

### 反面示范

| ❌ 模糊假设 | ✅ 精确假设 |
|-----------|-----------|
| "看看薪资和AI暴露度的关系" | "H1: 中国职业AI暴露度与薪资水平呈正相关（r>0.3, p<0.05）" |
| "分析就业趋势" | "H2: 高AI暴露职业的就业人数在2022-2024年间下降超过10%" |
| "比较不同群体" | "H3: 高暴露低薪群体（暴露度>0.7, 薪资<P50）占比超过20%" |

## 约束条件写法

明确限制分析范围，防止过度推断：

```markdown
## 约束条件

- **数据可信度优先**: 官方统计数据 > 平台报告 > 推测估算
- **抑制标记处理**: 样本量<30或缺失率>20%的子群体不纳入结论
- **结论范围声明**: 所有结论仅适用于[具体范围]，不推广到[范围外]
- **时间范围**: 分析基于[开始日期]至[结束日期]的数据
```

## 执行流程

1. **接收需求** → 识别用户的真实业务问题
2. **追问澄清** → 用5-10个问题填补信息缺口
3. **结构化输出** → 按四要素模板生成任务书
4. **确认对齐** → 与用户确认任务书理解正确
5. **进入执行** → 调用 data-quality-check 开始 Step1

## 禁止事项

- 不猜测用户意图：有模糊处必须追问确认
- 不省略假设验证：没有假设的分析是数据漫游
- 不扩大结论范围：明确标注分析的边界和局限性
- 不在步骤中标注工作时间（如"1天"、"2天"）：时间预估不准，且不属于任务书的分析内容

## 示例

### 输入（模糊需求）

> "我想分析一下 AI 对就业的影响"

### 输出（结构化任务书）

```markdown
# 分析任务书：AI 暴露度与就业市场分析

## 分析目标

回答核心问题：在中国劳动力市场中，AI 技术对不同职业的薪资和就业规模产生了什么影响？

决策价值：为个人职业规划和政策制定提供数据依据。

## 待验证假设

**H1: AI 暴露度与薪资水平正相关**
- 数据来源: pku-ai-exposure-index-2025.pdf + 年鉴1-31-分职业工资价位2023.xlsx
- 验证方法: Pearson 相关性分析
- 成功标准: r > 0.3, p < 0.05, n ≥ 100

**H2: 高暴露职业中低薪群体面临双重挤压**
- 数据来源: 合并后的职业暴露度×薪资数据
- 验证方法: 交叉分析，计算特定群体占比
- 成功标准: 高暴露低薪群体占比 > 15%

**H3: 高难度职业 vs 低难度职业的 AI 暴露度差异显著**
- 数据来源: 年鉴分职业数据 + AI 暴露度指数
- 验证方法: 独立样本 t 检验
- 成功标准: Cohen's d > 0.5, p < 0.05

## 分析步骤

Step1: 数据校验 → 验证记录数、缺失率、格式一致性
Step2: 描述性分析 → 各变量分布、极值检查
Step3: 假设验证 → 逐一验证 H1/H2/H3
Step4: 探索发现 → 开放性挖掘意外模式
Step5: 产物输出 → insights.md + report.md + dashboard.html

## 产物要求

- **insights.md**: 3条核心洞察，每条配图表和简短解释
- **report.md**: 完整分析报告（摘要→数据说明→假设验证→核心发现→局限性）
- **dashboard.html**: 暖色极简风格，适合社交媒体截图

## 约束条件

- 官方统计数据优先，平台报告标注可信度为"中"
- 匹配率<60%的职业代码不纳入主要结论
- 结论仅适用于346个有暴露度数据的职业
```
