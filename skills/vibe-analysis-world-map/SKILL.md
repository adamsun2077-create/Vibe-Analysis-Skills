# 世界地图构建

面向国家/地区层级分析任务的世界地图构建技能，兼顾分析表达、社媒传播和边界敏感场景。

## 触发条件

- 用户要做世界地图、全球地图、国家分布图、国别对比
- 用户提到气泡地图、分级着色图、拓扑地图、全球底图
- 需要把人口、TFR、城市化率、GDP、渗透率等国家级指标映射到地图
- 需要做适合截图传播的单页 HTML 地图
- 需要处理中国发布场景、边界敏感、底图合规等问题

## 先判断三件事

### 1. 传播场景

- **分析工作台**：可接受更复杂交互
- **报告截图**：强调信息层级和图例清晰
- **中国社媒**：优先考虑地图表达是否自然、稳妥、可传播

### 2. 地图精度

- **严格 GIS**：边界、底图、缩放、地理精度是核心
- **国家级叙事地图**：重点是国家比较，不是精确测绘
- **拓扑/叙事地图**：重点是讲故事，弱化边界技术细节

### 3. 指标映射

先明确：

```text
颜色 = 什么指标？
大小 = 什么指标？
标签 = 什么指标？
时间轴变化时，哪些视觉通道要同步变化？
```

**一条纪律：一个指标只占一个主视觉通道。**

## 地图模式选择

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 需要真实底图、边界、缩放定位 | GIS 地图 | 地理精度优先 |
| 需要国家整体颜色对比 | 分级着色图 | 适合单指标覆盖 |
| 需要同时表达率值 + 规模 | 气泡地图 | 颜色与大小分工明确 |
| 边界敏感、社媒传播、国家级叙事 | 拓扑叙事地图 | 更自然，避免陷入底图争议 |

### 默认推荐

如果满足以下任意条件，默认优先 **拓扑叙事地图**：

- 用户强调“好看”“适合社媒”“截图传播”
- 用户只关心国家维度比较，不关心精确地理测绘
- 存在边界敏感或底图来源不稳定问题
- 需要在一个图里同时放颜色、气泡、标签、时间轴

## 技术路线选择

| 技术 | 适用场景 | 不足 |
|------|---------|------|
| `ECharts` | 快速成图、常规国家对比、较低开发成本 | 定制拓扑叙事能力一般 |
| `D3 + SVG + TopoJSON` | 高度定制、拓扑布局、社媒叙事图 | 开发复杂度更高 |
| `Leaflet / MapLibre` | 精确底图、真实 GIS 交互 | 社媒叙事图容易显得重 |

### 经验法则

- **快**：先用 `ECharts`
- **美观 + 强定制**：用 `D3 + SVG`
- **必须真地图**：才用 `Leaflet / MapLibre`

## 视觉编码规范

### 推荐映射

```text
颜色(fill)   → 比率/指数类指标
大小(radius) → 总量/绝对规模
标签(text)   → 第三个关键指标或国家名
描边(stroke) → 状态标记/选中态/特殊分组
虚线(dash)   → 历史事件或告警状态
```

### 标签层级

气泡图优先使用双行标签：

```text
上：国家名称（小字）
下：关键数字（大字）
```

规则：

- 小气泡隐藏国家名，只保留核心数字或全隐藏
- 中大气泡显示双行标签
- 图例中必须写清楚：颜色看什么，大小看什么，标签看什么

### 时间轴纪律

如果地图有年份滑块或播放动画，**所有依赖年份的视觉变量必须同步变化**：

- 颜色
- 气泡大小
- 标签数字
- tooltip 内容
- 排名/注释（如有）

禁止只更新一个指标，导致图面逻辑失真。

## 中国发布场景的处理原则

### 严格 GIS 场景

- 优先使用合规、明确来源的底图
- 不要把多个底图源叠在一起，避免重影或边界冲突
- 不要把“手改边界线”当作默认方案

### 非严格 GIS 场景

如果用户的重点不是测绘，而是国家级叙事表达：

- 优先改成 **拓扑图 / 轮廓图 / 气泡图 / 叙事地图**
- 让“国家比较”成为主叙事
- 让边界问题退回到底图选择问题，而不是业务图层问题

### 核心原则

> 如果用户要的是“自然展示世界格局”，不是“法律意义上的边界测绘”，优先换表达方式，不要硬改轮廓线。

## 实施流程

### Step 1：定义目标

先输出一句话方案：

```text
这张图要表达的是 [国家级比较 / 地理分布 / 趋势演化]，
因此采用 [GIS / 分级着色 / 气泡 / 拓扑] 方案。
```

### Step 2：确认映射

明确：

- 主指标
- 辅指标
- 时间维度
- 默认聚焦区域（如东亚、欧洲）
- hover / click / reset 是否需要

### Step 3：选择骨架

推荐单文件 `index.html` 骨架：

```text
数据层
→ 比例尺与颜色函数
→ 地图初始化
→ 节点/国家构建
→ 渲染底图
→ 渲染图层
→ tooltip / 点击详情 / 时间轴
→ resize / focus / reset
```

### Step 4：处理地图数据

- TopoJSON 数字 `id` 转 ISO 时，统一先转 `Number`
- JS 对象键不要写前导零数字键，避免 `012` 这类键带来错误映射
- 国别显示名、交互 ID、填充 ID 可以拆开管理

### Step 5：做传播优化

- 去掉不必要底图纹理
- 减少图层重叠
- 保留单一主视觉
- 提升标签和 tooltip 可读性
- 让截图一眼读懂

## 输出要求

至少交付以下内容：

- `index.html`
- 图例（明确颜色/大小/标签）
- tooltip 或详情面板
- 如有时间轴，需支持拖动与播放
- 默认视角与 reset 机制

## 验证清单

- [ ] 没有底图重影或双层轮廓
- [ ] 图例与实际编码一致
- [ ] 国家节点没有明显错位到海上/南极
- [ ] 时间轴拖动时颜色、大小、标签同步变化
- [ ] tooltip 数值与画面一致
- [ ] 小气泡不会被强塞满标签
- [ ] 如果是中国发布场景，表达方式足够自然稳妥

## 禁止事项

- 不要一个指标同时占颜色和大小，除非用户明确要求
- 不要默认上真实 GIS 重底图
- 不要为边界敏感问题优先做“手改边界线”
- 不要忽略图例和标签层级
- 不要让时间轴只改颜色、不改大小和数字
