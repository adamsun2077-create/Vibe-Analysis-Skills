# 假设驱动分析

不强行得出结论的分析思路。

## 触发条件

- 需要验证预设假设（H1/H2/H3）
- 进行相关性/因果分析
- 需要区分验证性分析与探索性分析

## 核心原则

### 1. 假设先行

**在看数据之前写好假设，防止结论先行。**

```markdown
## 反面案例（结论先行）
❌ "数据显示薪资和AI暴露度正相关"
   → 可能是看到数据后编的故事

## 正面案例（假设先行）
✅ "H1: 中国职业AI暴露度与薪资水平呈正相关（r>0.3, p<0.05）"
   → 先提出假设，再用数据验证
```

### 2. 验证 vs 探索分开

| 阶段 | 目的 | 方法 | 产物 |
|------|------|------|------|
| Step3 验证 | 检验预设假设 | 统计检验、假设检验 | 验证结果（成立/不成立） |
| Step4 探索 | 发现意外模式 | 可视化、聚类、异常检测 | 待验证的新假设 |

**关键纪律**：
- 验证阶段只回答预设假设，不新增假设
- 探索发现必须在下一轮迭代中独立验证
- 禁止用探索发现直接支持业务决策

### 3. 假设不成立时的处理

**直接说明实际发现，反直觉结论反而更有传播力。**

```markdown
## 假设验证结果示例

**H1: AI暴露度与薪资正相关**
- 预期：r > 0.3
- 实际：r = 0.12, p = 0.08
- 结论：**假设不成立**，AI暴露度与薪资无显著相关

**反直觉发现**：
高AI暴露职业中，部分高薪（金融分析师）与部分低薪（数据录入员）
并存，导致整体相关性被抵消。需进一步分群分析。
```

## 统计检验规范

### 相关性分析

```python
# 必须标注样本量和显著性
from scipy import stats

r, p = stats.pearsonr(x, y)
n = len(x)

# 输出格式
print(f"r = {r:.3f}, p = {p:.4f}, n = {n}")

# 解读
if p < 0.05 and abs(r) > 0.3:
    interpretation = "显著中等相关"
elif p < 0.05:
    interpretation = "显著但弱相关"
else:
    interpretation = "无显著相关"
```

**报告格式**：
```
AI暴露度与薪资水平相关性：r = 0.39, p = 0.002, n = 346
→ 显著中等正相关（解释方差约15%）
```

### 组间比较

```python
from scipy import stats

# t检验
t_stat, p = stats.ttest_ind(group1, group2)
cohen_d = (group1.mean() - group2.mean()) / pooled_std

# 输出格式
print(f"t = {t_stat:.3f}, p = {p:.4f}, Cohen's d = {cohen_d:.3f}, n1 = {len(group1)}, n2 = {len(group2)}")
```

**效应量解读**：
| Cohen's d | 效应大小 |
|-----------|---------|
| 0.2 | 小 |
| 0.5 | 中 |
| 0.8 | 大 |

### 回归分析

```python
import statsmodels.api as sm

model = sm.OLS(y, X).fit()

# 关键指标
print(f"R² = {model.rsquared:.3f}")
print(f"Adj R² = {model.rsquared_adj:.3f}")
print(f"F = {model.fvalue:.3f}, p = {model.f_pvalue:.4f}")
print(f"n = {int(model.nobs)}")

# 系数表格
print(model.summary())
```

## 显著性标注规范

所有统计结果必须包含：

```markdown
| 指标 | 数值 | 样本量 | p值 | 结论 |
|------|------|--------|-----|------|
| Pearson r | 0.39 | n=346 | p=0.002 | **显著** |
| Cohen's d | 0.62 | n1=150, n2=196 | p<0.001 | **显著** |
```

**显著性标记**：
- p < 0.001：***
- p < 0.01：**
- p < 0.05：*
- p ≥ 0.05：ns（不显著）

## 分析流程

```
Step 3: 假设验证
├── 对每个假设 H1/H2/H3
│   ├── 选择适当统计方法
│   ├── 执行检验
│   ├── 记录统计量 + n + p
│   └── 判断是否成立
├── 汇总验证结果表
└── 标注需要探索的反直觉发现

Step 4: 探索性发现
├── 开放性数据探索
├── 记录有趣模式（不作为结论）
├── 形成新假设（H4/H5...）
└── 建议下一轮验证
```

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| p值显著就说"证明"了假设 | 说"支持"或"不拒绝"，保留不确定性 |
| 只看p值忽略效应量 | 同时报告Cohen's d或R² |
| 多重比较不校正 | 使用Bonferroni或FDR校正 |
| 小样本显著性过度推广 | 明确标注样本量，谨慎外推 |
| 用探索发现直接做结论 | 标记为"待验证假设"，下一轮检验 |

## 输出模板

```markdown
# 假设验证结果

## H1: [假设描述]
- **验证方法**: [统计方法]
- **统计结果**: [r/t/F] = [值], p = [值], n = [值]
- **结论**: [成立/不成立/部分成立]
- **业务含义**: [一句话解释]

## H2: ...

## 探索发现（待验证）
1. [发现描述] → 建议作为H4在下一轮验证
2. ...

## 分析方法局限性
- [样本限制]
- [方法假设]
- [外推边界]
```

## 禁止事项

- 不进行多重比较校正：同时检验多个假设时必须校正
- 不报告样本量：所有统计结果必须标注n=
- 不区分验证与探索：探索发现不能作为验证结论
- 忽视效应量：p<0.05但效应量极小可能没有实际意义
