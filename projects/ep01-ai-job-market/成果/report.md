# AI 冲击中国职业市场：完整研究报告

**版本**：v1.0 | **日期**：2026-03-22
**研究框架**：沿用 Anthropic Economic Index 方法论，替换为中国本土数据

---

## 摘要

本研究基于中国 LLM 职业暴露指数、《中国劳动统计年鉴 2024》及 Anthropic Economic Index，对中国劳动力市场 AI 冲击程度进行实证分析。核心发现：

1. **中国整体符合全球规律**：高薪知识型职业 AI 暴露度更高（Spearman ρ = 0.53），与 Anthropic 全球发现一致
2. **中美存在系统性认知差异**：中国高估管理类职业风险、低估销售/接待/翻译类风险（ρ = 0.385 < 0.7）
3. **制造业是最大规模潜在冲击源**：3577 万从业者处于中等暴露区间，影响规模指数全行业第一
4. **12 个中美共识高危职业**：数据库架构师、会计师、财务分析师、软件工程师等

### 1.2 方法论差异（重要）

中国指数与 Anthropic 指数**不可直接比较绝对数值**：

| 维度 | 中国 LLM 暴露指数 | Anthropic 暴露指数 |

| 解读含义 | 职业任务理论上能被 LLM 完成的比例 | 实际已有多少比例在使用 AI 完成 |

**比较策略**：所有中美对比仅使用职业排名（谁高谁低），不使用绝对数值。

## 二、数据处理逻辑

### 2.1 整体处理流程

```
原始数据（4个来源）
    │
    ├── 中国 LLM 暴露指数（Excel）
    │       ↓ 读取 + SOC 代码标准化
    │   exposure_base_soc_detail.xlsx（100条 SOC 细分）
    │   exposure_base_minor_soc.xlsx（60条 SOC 组别，含中文名）
    │   exposure_base_zl_occu.xlsx（100条智联职业）
    │
    ├── Anthropic 暴露指数（CSV）
    │       ↓ 直接读取，无需清洗
    │   anthropic-job-exposure.csv（756条）
    │
    ├── 年鉴工资数据（Excel 宽表）
    │       ↓ 跳过标题行 → 提取职业名称 + P10/P25/P50/P75/P90
    │   61个职业中类 × 5个工资分位数
    │
    └── 年鉴就业数据（Excel 宽表）
            ↓ 跳过标题行 → 提取19个行业 + 全国合计就业人数
        19个行业大类 × 就业人数（万人）
            │
            ▼
    ┌─────────────────────────────────────────────────────┐
    │                   三条分析路径                        │
    ├─────────────────────────────────────────────────────┤
    │ H1路径：年鉴工资 × SOC暴露指数                        │
    │ H2路径：中国SOC暴露指数 × Anthropic暴露指数            │
    │ H3路径：年鉴就业人数 × 智联行业暴露指数                 │
    └─────────────────────────────────────────────────────┘
```

---

### 2.2 H1 数据处理：年鉴工资 × SOC 暴露指数

**步骤一：年鉴工资数据清洗**

原始 Excel 前 3 行为标题和单位说明，从第 4 行（index=4）起为有效数据，至第 65 行（index=64）结束，共 61 个职业中类。

```
原始列结构（header=None）：
  col[0] = 职业名称
  col[1] = P10（万元/年）
  col[2] = P25
  col[3] = P50（中位数）
  col[4] = P75
  col[5] = P90
```

**步骤二：职业中类 → SOC minor 组别映射（人工规则）**

年鉴职业中类与 SOC 职业体系存在粒度差异，采用业务语义规则做人工映射。关键约束：同一 SOC 组别可对应多个年鉴职业（多对一），但同一年鉴职业只映射至一个 SOC。

映射覆盖率：61/61 = 100%，其中涉及的独立 SOC 组别共 **28 个**（多对一导致重复使用 8 个 SOC 代码）。

**步骤三：多对一问题的处理**

发现问题：51-9000（其他生产职业）被 15 个年鉴职业共用，工资范围 4.97–7.59 万，但暴露度统一为 0.451，造成统计噪音。

```
处理方式：
  全量计算（n=61）：含重复SOC，ρ = 0.202 → 不采用
  去重计算（n=28）：每个SOC仅保留第一条，ρ = 0.530 → 主要结论
```

主要结论基于去重后 28 个独立 SOC 类别，并在报告中注明映射粒度局限。

---

### 2.3 H2 数据处理：中国 SOC 暴露指数 × Anthropic 暴露指数

**步骤一：SOC 代码格式标准化**

两套数据的 SOC 代码格式不一致，需统一后做精确匹配：

```
中国格式：11-1011.00（带 .00 后缀）
Anthropic：11-1011（无后缀）
处理：中国代码去掉 .00 → 统一为 XX-XXXX 格式
```

**步骤二：精确匹配合并**

```
中国 SOC 细分（100条）× Anthropic（756条）
精确匹配（inner join on SOC代码）= 91条
未匹配 9 条（中国有但Anthropic无的SOC代码）
```

**步骤三：排名计算**

两套指数尺度不同（中国 0.17–0.91，Anthropic 0.00–0.75），仅在合并后的 91 条样本内分别计算排名：

```
cn_rank：对 exposure 降序排名（1=最高暴露）
ant_rank：对 observed_exposure 降序排名（1=最高暴露）
delta_rank = cn_rank - ant_rank
  负值 → 中国排名更靠前（中国认为更危险）
  正值 → 全球排名更靠前（全球认为更危险）
```

**步骤四：0 值处理**

Anthropic 数据中有 411/756（54.4%）职业暴露度 = 0。在 91 个匹配职业中，有 18 个 Anthropic 暴露度 = 0，统一获得并列排名 82.5（tied rank 平均值）。

```
影响：涉及0值的 delta_rank 不反映真实认知分歧，仅反映"Anthropic无使用记录"
处理：所有结论引用的差异案例，限于 Anthropic 暴露度 > 0 的 73 个职业
```

---

### 2.4 H3 数据处理：年鉴就业人数 × 智联行业暴露指数

**步骤一：年鉴就业数据清洗**

原始 Excel 为宽表，行为统计口径，列为行业，需定位正确的数据行：

```
行结构：
  index[3]：列标题行（19个行业名称）
  index[5]：2023年全国合计数据行
  取 col[2:21]（19个行业）及 col[1]（全国合计）
口径：城镇非私营单位就业人员年末人数（万人）
```

**步骤二：智联行业暴露指数聚合**

智联数据含 100 个职业细类（jd_class2），按 jd_class1 取均值，得到 19 个行业的代表性暴露指数：

```
exposure_by_industry = groupby('jd_class1')['exposure'].mean()
```

**步骤三：行业维度映射（人工规则）**

年鉴行业分类（国民经济行业分类）与智联 jd_class1 为不同维度体系，采用业务语义规则对应：

```
可映射：17/19 个行业（89.5%，覆盖 14,053 万人）
无法映射：
  - 采矿业（329万）：智联无此行业分类
  - 公共管理/社会保障（1986万）：非市场化招聘，智联无覆盖
```

**步骤四：影响规模指数计算**

```
影响规模指数 = 就业人数（万）× 暴露指数
含义：综合衡量"规模×强度"的潜在冲击量级，非标准统计指标
```

---

### 2.5 关键设计决策说明

| 决策点 | 选择 | 原因 |
|--------|------|------|
| 中美比较方式 | 仅比较排名，不比较数值 | 两套指数方法论不同，绝对值无可比性 |
| H1 去重策略 | 每个 SOC 取第一条 | 消除多对一映射带来的统计噪音 |
| H2 0值过滤 | 结论仅引用非0值样本 | 0值反映"无使用记录"，非真实低风险 |
| H3 分析粒度 | 行业大类（17条） | 行业→职业维度不兼容，粗粒度映射误差更可控 |
| 影响规模指数 | 就业人数 × 暴露指数 | 同时体现"谁被影响"和"影响多少人"，非正式统计指标 |
| Spearman vs Pearson | Spearman（基于排名） | 暴露指数分布非正态，排名相关更稳健 |

### 3.3 关键异常点

**异常一：采矿人员**（P50 = 7.59 万，暴露度 = 0.451）
- 工资排名前 15%，但暴露度排名后 40%
- 解释：采矿工作高度依赖物理环境感知和操作，LLM 无法替代核心任务

**异常二：行政文员类**（P50 = 6.60 万，暴露度 = 0.720）
- 工资中等偏下，但暴露度极高
- 解释：文书处理、信息整理等任务正是 LLM 的强项，属于"低薪高暴露"的脆弱群体

## 六、综合发现与探索性结论

### 6.1 双重脆弱职业

高暴露 + 中低工资（最需关注的群体）：

| 职业 | P50 工资 | 暴露度 | 脆弱特征 |
|------|---------|-------|---------|
| 文化和教育服务人员 | 6.17 万 | 0.786 | 暴露极高，工资中等偏低 |
| 行政文员 | 6.52 万 | 0.724 | 暴露高，工资中等 |
| 租赁和商务服务人员 | 4.68 万 | 0.693 | 暴露高，工资偏低 |

### 6.2 中美差异的可能结构性原因

**管理类职业中美分歧（中国高估）**：
- 中国招聘 JD 中，管理岗位通常包含大量具体技术职责描述（如 IT 经理往往写明编程技能要求），导致任务-LLM 匹配度虚高
- Anthropic 真实数据显示：实际使用 Claude 的管理者，更多用于文档撰写而非核心管理决策

**销售类职业中美分歧（中国低估）**：
- Anthropic 数据显示销售类职业大量使用 Claude 辅助撰写提案、邮件、客户沟通材料
- 中国销售岗 JD 更偏重关系维护和渠道开拓，任务描述中文字工作占比较少，导致暴露度被低估

## 七、局限性声明

1. **时间错位**：中国暴露指数基期为 2018–2021，工资/就业数据为 2023，AI 能力演进期间职业风险已发生变化
2. **数据口径**：就业数据为城镇非私营单位，不含私营企业，制造业等实际从业规模被低估
3. **因果限制**：所有结论为相关关系，无法推断 AI 是否"导致"就业或工资变化
4. **覆盖范围**：分析基于能成功匹配的职业（SOC 91 个 / 年鉴 61 类），未覆盖职业不作推论
5. **平台偏差**：中国暴露指数来自智联招聘，系统性低估非市场化就业（政府机关、国企）
